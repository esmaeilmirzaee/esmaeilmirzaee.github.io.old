Enabling proxy setting on Docker CE

Post-installation

To verify docker group is created, use the following command:

$ cat /etc/group
or 
$ getent group
Unless docker group does not exist, run 
sudo groupadd docker

Replace user-name with your username. You can get infromation about your username by prompting whoami on the command line. 

sudo usermod -aG docker "user-name"

How to set proxy for Docker CE

If your machine/server is behind a proxy server, in order to use Docker CE you should follow a few more steps. Configuring Docker CE is straightforward task. Well lets start:

1. create a systemd drop-in directory for Docker: 
mkdir -p /etc/systemd/system/docker.service.d

2. Create a file or open a file named 'http-proxy.conf' and/or 'https-proxy.conf' if you need 'http' and/or 'https', repectively:

2.1. http 
touch /etc/systemd/system/docker.service.d/http-proxy.conf
2.2. https 
touch /etc/systemd/system/docker.service.d/https-proxy.conf

3. Add the following lines into the proper configure file:

3.1. Configuring proxy behind a https server 
[Service]
        Environment="HTTP_PROXY=http://proxy.example.com:80/"
3.2. Configuring proxy behind a http server 
[Service]
        Environment="HTTPS_PROXY=https://proxy.example.com:80/"

4. Enabling the latest altering

$ sudo systemctl daemon-reload

5. Rebooting the Docker deamon:

$ sudo systemctl restart docker

6. Verifying the Environment variables

systemctl show --property=Environment docker
